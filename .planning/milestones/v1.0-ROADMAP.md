# Roadmap: PBTES Simulation Pipeline

## Overview
A journey to stabilize the PBTES simulation, integrate Molten Salt as the single HTF, and automate a robust parametric sweep pipeline for a Q1 academic article.

- [x] **Phase 1: Codebase Stabilization** - Clean up the codebase, fix silent bugs, and establish robust foundations (completed 2026-04-22)
- [x] **Phase 2: Convergence & Metrics** - Map thermodynamic constraints for 98% convergence, improve control logic, and calculate metrics (completed 2026-04-22)

## Phase Details

### Phase 1: Codebase Stabilization
**Goal**: Clean up the codebase, fix silent bugs, and establish robust foundations
**Depends on**: Nothing
**Requirements**: [CORE-01]
**Success Criteria** (what must be TRUE):
  1. Bare exceptions are replaced with explicit error handling and logging.
  2. The core simulation runs without silent failures.

### Phase 2: Convergence & Metrics
**Goal**: Map thermodynamic constraints for 98% convergence, improve control logic, and calculate metrics
**Depends on**: Phase 1
**Requirements**: [CORE-02, CORE-03, METRICS-01]
**Success Criteria** (what must be TRUE):
  1. The solver converges ≥98% of the time under standard steady and quasi-steady test conditions.
  2. Control logic maximizes solar fraction instead of just using naive temperature limits.
  3. Pump power is dynamically calculated (either via pressure modeling or mass-flow post-processing).
  4. Plant Factor and Solar Fraction are accurately computed.
**Plans**: 2 plans

Plans:
- [x] 02-01-PLAN.md — Improve control logic and solver convergence
- [x] 02-02-PLAN.md — Dynamically calculate pump power and other plant metrics

### Phase 3: Architecture Upgrades
**Goal**: Introduce Molten Salt as the unified HTF and finalize Parallel/Series configs
**Depends on**: Phase 2
**Requirements**: [ARCH-01, ARCH-02, ARCH-03]
**Success Criteria** (what must be TRUE):
  1. The system correctly simulates using Molten Salt.
  2. Heat exchangers inside the PBTES loop are successfully removed.
  3. Both Parallel and Series configurations are selectable and function correctly.
**Plans**: 1 plan

Plans:
- [x] 03-01-PLAN.md — Unify HTF, replace HXs with Pipes, and implement Series/Parallel topologies

### Phase 4: Parametric Pipeline & Analysis
**Goal**: Build automated sweep scripts, generate graphs, integrate LCOH, and synthesize narrative
**Depends on**: Phase 3
**Requirements**: [ANALYSIS-01, ANALYSIS-02, ANALYSIS-03, ECON-01]
**Success Criteria** (what must be TRUE):
  1. An automated script runs multiple parameter sets and compiles the results.
  2. LCOH is calculated for each parametric run based on CAPEX/OPEX.
  3. Publication-quality (Q1) graphs are generated for single and parametric runs.
  4. A synthesis document outlines the results narrative for the article.
**Plans**: 2 plans

Plans:
- [x] 04-01-PLAN.md — Build parametric sweep script and integrate dynamic LCOH calculations
- [x] 04-02-PLAN.md — Generate publication-quality plots and synthesize results into Markdown
